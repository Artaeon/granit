# Map of Content — Knowledge Management

This is a **Map of Content (MOC)** that organizes notes related to knowledge management, personal productivity, and the tools and techniques for managing information effectively.

## What is a Map of Content?

A MOC is a note that serves as an index or hub for a topic area. Unlike a rigid folder hierarchy, MOCs use links to create flexible, overlapping categorizations. A note can appear in multiple MOCs without duplication.

> Think of MOCs as curated playlists for your knowledge.

### Methodology
The approach used in this vault follows the **Zettelkasten** principles:
1. **Atomic notes** — Each note covers one idea or topic
2. **Connections over categories** — Links matter more than folders
3. **Emergence** — Structure emerges from connections, not top-down planning
4. **Write to think** — The act of writing clarifies understanding

## Principles for This Vault

1. **Link liberally.** When in doubt, create a `[[wikilink]]`. The graph view reveals surprising connections.
2. **Tag consistently.** Use lowercase, hyphenated tags in frontmatter (`machine-learning`, not `ML`).
3. **Review regularly.** The [[Templates/Weekly Review]] template keeps the vault alive.
4. **Prefer flat structure.** Use folders sparingly — MOCs provide better organization than deep hierarchies.
5. **Write for your future self.** Add enough context that you can understand the note six months later.
